RoundToEqualLevels(Double, Double, Int32)

Basically helper for when you need to choose 0..N-1 element based on what percentage does actual/max takes. Example: We have a stack of 30 max elements. When actual is:

  • 0..9 we return 0.
  • 10..19 we return 1.
  • 20..30 we return 2.

Useful when selecting N sprites for display in stacks, etc.

RoundToLevels(Double, Double, Int32)

Assigns the value actual going from 0 to max such that it is divided into a set of (amount levels) "levels". Rounding is performed to the "middle" so that the highest and lowest levels are only assigned if actual is exactly max or 0.

Examples

Say you have a progress bar going from 0 -> 100 inclusive and you want to map this to 6 sprite states (0, 4 intermediates and full). This method allows you to easily map this progress bar to the sprite states.

RoundToNearestLevels(Double, Double, Int32)

Returns the segment actual lies on on a decimal scale from 0 to max divided into levels sections. In less mathematical terms, same as RoundToLevels(Double, Double, Int32) except actual is rounded to the nearest matching level instead of 0 and the highest level being precisely 0 and max and no other value.

Examples

You have a 5-segment progress bar used to display a percentile value. You want the display to match the percentile value as accurately as possible, so that eg. 95% is rounded up to 5, 89.99% is rounded down to 4, 15% is rounded up to 1 and 5% is rounded down to 0, in terms of number of segments lit. In this case you would use 

RoundToNearestLevels(value, max, 5)
